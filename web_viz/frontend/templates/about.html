About

In recent years, Deep Neural Networks have become an essential tool for many machine learning tasks, especially image classification. However, this has given rise to the study of adversarial samples - carefully crafted inputs for a neural network that result in an incorrect classification.

In the spirit of the TensorFlow Playground, we present the TextAttack Webapp: a web-based visualization tool to assist users in understanding the differences between common adversarial machine learning techniques. This application is based off of a previous project from the QData group: Andrew Norton's Adversarial Playground, and Jennifer Fang's Adversarial Playground on text (with only one algorithm). TextAttack Webapp makes a number of additions and improvements to help better educate and interact with users about how adversarial examples work. This project adds three new goals of user-friendliness, visual aid, and extendibility to Nortonâ€™s original goals for Adversarial-Playground to be educational, modular, interactive.


About the Attacks


Algorithm
The core aim of this webapp is to show how to generate efficient adversarial samples on text input. Many possible algorithms can attack text sequence inputs in a black-box manner. TextAttack presents variations of scoring strategies, outlined below, to identify critical tokens in a text sequence input that, if modified, can cause classifiers to incorrectly classify inputs with a high probability. Simple word or character-level transformations are applied to those highest-ranked critical tokens to minimize the differences, also known as the edit distance, between the original input and the generated adversarial example. For most effective usage, please input a text sequence of at least 5 words.
All outputs will be displayed in lowercase

Notable features of the TextAttack are the model, power, transformer algorithm, and scoring algorithm.

  • Model:

    This describes the underlying dataset being used in the TextAttack algorithm.
    (1) Amazon Review (Polarity): Polarity is more simplistic than the Full system since it only classifies inputs as Negative or Positive
    (2) Yelp Review (Full): 1-5 Stars

  • Power:

    The number of words per sequence that will be modified. If this number is higher than the length of the text sequence, then all the words in the sequence will be modified.

  • Scoring Algorithm:

    (0) Random (baseline): This scoring function randomly selects tokens as targets. In other words, it has no method to determine which tokens to attack.
    (1) Replaceone: Score of the sequence with and without the token.
    (2) Gradient (baseline): Contrary to random selection which uses no knowledge of the model, we also compare to full knowledge of the model, where gradients are used to find the most important tokens.

  • Transformer Algorithm:

    (1) character level, like Homoglyph: Replace character with a similar-looking, yet different character. This is the best performing transformer.
